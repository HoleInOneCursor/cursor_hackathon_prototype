# cursor_hackathon_prototype

## How agents run this repo

This repository currently contains documentation only. Agents do not need to
install dependencies, start services, or run build steps before making changes.

Recommended agent workflow:

1. Inspect the current branch and working tree with `git status`.
2. Read the relevant documentation files before editing.
3. Make focused changes and review the diff with `git diff`.
4. No automated test suite is configured yet; for README-only changes, validate
   by reviewing the rendered Markdown or the plain-text diff.
5. Commit and push changes to the task branch, then open a pull request.

If source code, package manifests, or test configuration are added later, update
this section with the exact setup, run, and test commands agents should use.
