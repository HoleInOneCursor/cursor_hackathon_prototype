# cursor_hackathon_prototype

## Running this repo with agents

This repository is currently documentation-only, so agents do not need to install dependencies or run a build before making changes.

Recommended agent workflow:

1. Confirm the assigned git branch and check the working tree with `git status`.
2. Read the README before editing so changes fit the existing documentation.
3. Make focused documentation updates directly in `README.md`.
4. Review the diff and confirm there are no unintended changes.
5. Commit the update with a descriptive message and push the assigned branch.
6. Open a pull request that summarizes the documentation change.

If code, tests, or tooling are added later, update this section with the required setup and verification commands so future agents can run the repo consistently.
