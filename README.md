# cursor_hackathon_prototype

## Agent workflow

This repository currently contains project documentation only. Agents working in
this repo should use the standard GitHub workflow below.

1. Work on the assigned feature branch for the task. For this task, use
   `cursor/readme-agent-instructions-c664`.
2. Inspect the repository before making changes so edits stay scoped to the
   existing files and conventions.
3. Make focused changes and verify them locally. For documentation-only changes,
   review the rendered Markdown or inspect the diff; there is no build or test
   command configured in this repository yet.
4. Commit with a clear message, then push with:

   ```sh
   git push -u origin cursor/readme-agent-instructions-c664
   ```

5. Open or update a GitHub pull request against `main`. Use a descriptive title
   and summary, and mark the PR ready for review unless the task explicitly asks
   for a draft.
